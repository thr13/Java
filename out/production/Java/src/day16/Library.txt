<라이브러리>
라이브러리는 프로그램 개발시, 활용가능한 클래스와 인터페이스들을 모아놓은 것을 의미한다
일반적으로 JAR(Java Archive) 압축파일(*.jar) 형태로 존재한다
JAR 파일에는 클래스, 인터페이스의 바이트코드 파일(*.class) 들이 압축되어 있다

그러므로 응용 프로그램을 개발할 때 자주 사용되는 공통인 클래스, 인터페이스를 JAR 파일로 압축해서 라이브러리로 관리하는 것이 좋다

라이브러리를 이용하려면 JAR 파일을 ClassPath 에 추가해야 한다
-> 터미널 기준으로 java 명령어를 실행할때, -classpath 로 제공 또는 CLASSPATH 환경 변수에 경로 추가

인텔리제이 기준으로
JAR 파일 생성 방법)
프로젝트구조(cmd+;)-아티팩트-추가-JAR-경로지정 후 빌드-모듈빌드
외부 라이브러리 사용 방법)
프로젝트구조(cmd+;)-모듈-종속속성-추가-JAR 또는 디렉토리-경로에서 JAR 파일을 설정 후 확인
