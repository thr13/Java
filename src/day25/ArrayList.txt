ArrayList 는 java.util 패키지에 존재함

컬렉션 프레임워크(Collection Framework)는 List, Set, Map 등의 인터페이스를 통해서 컬렉션 클래스를 이용할 수 있도록 설계되어 있다
List 인터페이스는 순서가 존재하며 중복 저장이 가능하다 구현클래스로는 ArrayList, Vector, LinkedList 가 있다

ArrayList 는 내부 배열에 객체를 저장한다
일반 배열과 달리 제한 없이 객체를 추가할 수 있다

읿반적으로 List 컬렉션은 객체 자체를 저장하는게 아니라 객체의 번지(메모리주소)를 저장한다
또한, 동일한 객체를 중복 저장할 수 있다 이 경우는 객체가 저장된 동일한 번지가 저장된다 (+ null 도 저장가능)

ArrayList 는 인덱스 0번 부터 차례대로 저장된다
특정 인덱스의 객체 제거시 바로 뒤 인덱스부터 끝까지 1씩 당겨진다
특정 인덱스의 객체 추가시 바로 뒤 인덱스부터 끝까지 1씩 밀려난다

* 객체 삽입/삭제가 빈번할 경우, ArrayList 는 비효율적이다 (이 경우는 LinkedList 사용)

ArrayList 는 둘 이상의 스레드가 동시에 메소드 호출이 가능하다(비동기적)
이 경우 메소드가 경합이 발생해 하나만 저장될 수 있어 손실이 발생한다
