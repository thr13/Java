자바 프로그램은 운영체제 위에서 바로 실행되는게 아니라 JVM(자바 가상 머신) 위에서 실행된다
따라서 운영체제의 모든 기능을 자바 코드로 직접 접근하기 어렵다
그러나 java.lang 패키지에 소속된 System 클래스를 이용하면 운영체제의 일부 기능을 이용할 수 있다

운영체제는 실행 중인 프로그램을 프로세스(process)로 관리한다
자바 프로그램을 시작하면 JVM 프로세스가 생성되고 이 프로세스가 main() 메소드를 호출한다

시스템 프로퍼티(System Property)란, 자바 프로그램이 시작될 때 자동 설정되는 시스템의 속성을 의미한다
예) 운영체제 종류, 사용자 정보, 자바 버전 등의 기본 사양 정보
시스템 프로퍼티의 주요 속성 이름과 값
java.specification.version / 자바 스펙버전 / 17
java.home / jdk 디렉토리 겨올 / C:\~~~
os.name / 운영체제 / Windows 10
user.name / 사용자 이름 / ~~~
user.home / 사용자 홈 디렉토리 경로 / C:\Users\~~~
user.dir / 현재 리덱토리 경로   / C:/프로젝트경로~~~

System 클래스의
정적멤버:
    out - 콘솔에 문자 출력
    err - 콘솔에 에러 내용 출력
    in - 키보드 입력
메소드:
    exit(int status) - 프로세스 (강제) 종료, 여기서 매개변수인 int status 은 종료 상태값 이다 종료 상태값으로 어떤 값을 주더라도 프로세스는 종료되는데 정상종료는 0 비정상종료는 1 또는 -1 을 주는게 관례이다
    currentTimeMillis() - 현재 시간을 밀리초(ms, 1000 분의 1) 단위의 long 값으로 리턴 / 1970년 1월 1일 0시부터 시작해서 현재까지 진행된 시간을 반환함
    nanoTime() - 현재 시간을 나노초(ns, 10^9 분의 1) 단위의 long 값으로 리턴 / 1970년 1월 1일 0시부터 시작해서 현재까지 진행된 시간을 반환함
    getProperty() - 운영체제와 사용자 정보 제공
    getenv() - 운영체제의 환경 변수 정보 제공
