운영체제(OS, Operating System) 은 실행 중인 프로그램을 프로세스(process)로 관리함
멀티태스킹(multi tasking) 은 2 가지 이상의 작업을 동시에 처리하는 것을 의미
멀티태스킹시 운영체제는 멀티 프로세스를 생성해서 작업을 처리함 (하지만 항상 멀티 태스킹이 멀티 프로세스를 의미하진 않는다)

하나의 프로세스 내에서 멀티 태스킹도 가능 -> 대표적인 예로 메신저 프로그램이 있다 (채팅과 동시에 파일 전송 작업 수행)

스레드(thread)는 코드의 실행 흐름을 의미
스레드는 프로세스 내부에서 생성됨

하나의 프로세스가 2가지 이상의 작업을 처리할 수 있는 이유는 멀티 스레드(multi thread)가 있기 때문
프로세스 내부에 스레드가 2개 라면, 2개의 코드의 실행 흐름이 생긴다는 의미

멀티 프로세스 == 프로그램 단위의 멀티 태스킹
멀티 스레드 == 프로그램 내부의 멀티 태스킹

멀티 스레드는 서로 독립적 -> 하나의 프로세스에서 오류가 발생해도 다른 프로세스에게 영향을 미치지 않음
그러나 멀티 스레드는 프로세스 내부에서 생성되므로 하나의 스레드가 예외를 발생시키면 해당 프로세스가 종료되므로 같은 프로세스 내 다른 스레드에게 영향을 미친다
그렇기 때문에 멀티 스레드 사용시 예외 처리를 주의해야 한다

멀티 스레드는 다양한 분야에 활용된다
데이터를 분할해서 병렬로 처리하는 곳에서 사용되거나
안드로이드 앱에서 네트워크 통신을 하기 위해서 사용되거나
다수의 클라이언트 요청을 처리하는 서버를 개발할때 등등 사용된다

JAVA 21 버전부터 가상 스레드(virtual thread) 기능이 추가되었다
여기서의 스레드(thread) == 운영체제 스레드와 1:1 로 매핑되는 플랫폼 스레드(platform thread)일 때,
복수 개의 가상 스레드가 운영체제 스레드 1개와 매핑되면서 실행됨
가상 스레드 사용시 제한된 운영체제의 스레드를 효율적 사용하면서 여러 작업을 동시에 처리하는 동시성 애플리케이션의 속도를 향상시킬 수 있다
